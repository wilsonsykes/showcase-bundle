Brand Hub identity

Top-of-page details for your Brand Hub (the Linktree-style site). The name is shown as the big header, the tagline as the small line below, and the logo is an optional square image. Placeholder until you finalize the hub name.

Sub-brands

One row per sub-brand under the umbrella. Each brand gets its own page on the hub, reached by tapping its button on the home screen. The Key is the internal id (no spaces, lowercase, e.g. quadro-decor) — it's how links below reference their brand. Accent is the color used on that brand's button and page. Drag rows to reorder how brands appear on the home page.

Links

Each link is assigned to a sub-brand. The Category field controls where the link appears:

  • Category = materials → appears in the Materials section on the home page (the brand name auto-prefixes the subtitle so visitors know whose it is)
  • Any other category (or blank) → appears inside that brand's drill-down page
  • Umbrella-wide materials: leave the Brand column blank when the file covers the whole company (e.g. a corporate brochure). The subtitle will auto-prefix with your umbrella brand name instead of a sub-brand. Only allowed when category = materials.

Drag rows to set order. category is also future-proofed for filter pills inside brand pages later when one brand grows past ~8 links.

Showcase data folder Not connected

Pick the data/ folder inside your showcase-site/ folder once. After that, Save to showcase folder writes products.json and categories.json directly. Refresh the showcase in your other tab to see changes.

Brand Hub data folder Not connected

Pick the data/ folder inside your brand-hub/ folder once. Save to hub folder writes links.json (containing both brand info and the link list). Refresh the hub to see changes.

Browser doesn't support direct file saving. The File System Access API is available in Chrome, Edge, Brave, and Opera. In Safari and Firefox, use Download data files below and replace files manually.

API sync (Postgres) Not configured

Use this to sync data directly with the local API and Postgres. Keep folder save/download as fallback. Default API base is http://localhost:4100.

Download data files

Alternative if you can't (or don't want to) connect folders. Downloads the JSON files. Drop products.json and categories.json into showcase-site/data/, and links.json into brand-hub/data/.

Full backup

Save all your dashboard data (products, categories, brand info, hub links) as one JSON file you can keep on Dropbox, Google Drive, or anywhere safe. Restore from that file later or on another computer.

Reset everything

Wipes all data from this browser. Useful if you want to start fresh or imported the wrong data. Always export a backup first.

Where your data lives: Everything you edit here is stored in this browser's local storage on this computer. It does not sync between computers or browsers, and clearing your browser data will wipe it. Export to JSON regularly as a real backup, especially before clearing browser cache or moving to a new device.
